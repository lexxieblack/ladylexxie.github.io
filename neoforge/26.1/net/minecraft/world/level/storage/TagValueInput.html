    • rawChildOrEmpty

      public ValueInput rawChildOrEmpty(String key)
      Description copied from interface: ValueInputExtension
      Reads the child object from the given key, or provides an empty ValueInput if the child does not exist. This function behaves differently than ValueInput.childOrEmpty(String) in the case where the child exists (as reported by ValueInputExtension.keySet()) but is empty. While ValueInput.childOrEmpty(String) will return an empty ValueInput from which nothing can ever be read with a codec, this function will instead return a a wrapper around the empty child, allowing values to be read with a codec, provided that the codec can deserialize empty map-like objects.
      If not implemented, defaults to ValueInput.childOrEmpty(String).
      Specified by:
      rawChildOrEmpty in interface ValueInputExtension
      Parameters:
      key - the key to read the child from
      Returns:
      the ValueInput of the child, or an empty input

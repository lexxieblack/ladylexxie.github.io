Class DisplayWindow

java.lang.Object
net.neoforged.fml.earlydisplay.DisplayWindow
All Implemented Interfaces:
EarlyLoadingScreenController, ImmediateWindowProvider
public class DisplayWindow extends Object implements ImmediateWindowProvider
The Loading Window that is opened Immediately after Forge starts. It is called from the ModDirTransformerDiscoverer, the soonest method that ModLauncher calls into Forge code. In this way, we can be sure that this will not run before any transformer or injection.

The window itself is spun off into a secondary thread, and is handed off to the main game by Forge.

Because it is created so early, this thread will "absorb" the context from OpenGL. Therefore, it is of utmost importance that the Context is made Current for the main thread before handoff, otherwise OS X will crash out.

Based on the prior ClientVisualization, with some personal touches.

    • initWindow

      public void initWindow()
      Called to initialize the window when preparing for the Render Thread.

      The act of calling glfwInit here creates a concurrency issue; GL doesn't know whether we're gonna call any GL functions from the secondary thread and the main thread at the same time.

      It's then our job to make sure this doesn't happen, only calling GL functions where the Context is Current. As long as we can verify that, then GL (and things like OS X) have no complaints with doing this.
